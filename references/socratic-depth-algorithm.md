# 苏格拉底式追问深度算法

## 深度层级定义

| 深度 | 名称 | 追问目标 | 孩子当前状态要求 |
|:----:|------|----------|------------------|
| **L0** | 诊断层 | 确定卡点位置 | 刚接手、状态未知 |
| **L1** | 感知层 | 让孩子先有一个起点 | 需要最低门槛的提示 |
| **L2** | 推理层 | 让孩子讲出"为什么" | L1 通过，能给出正确答案 |
| **L3** | 迁移层 | 让孩子自己变题或找不同方法 | L2 通过，能讲清算理 |
| **L4** | 创造层 | 让孩子出题或预测 | L3 通过，能变题或一题多解 |

## 年级深度上限

| 年级段 | 最大深度 | 原因 |
|--------|:--------:|------|
| **1-2年级** | L2 | 形式运算阶段未到，迁移和创造需具体实物支撑 |
| **3-4年级** | L3 | 开始能举一反三，但跨领域迁移仍吃力 |
| **5-6年级** | L4 | 抽象思维初步成熟，可以尝试创造层 |

## 动态升降级规则

| 孩子表现 | 系统响应 | 深度变化 | 连续计数 |
|----------|----------|:--------:|:--------:|
| ✅ 给出正确答案，并能说出理由 | 升级追问 | L → L+1（不超过年级上限） | fail_count = 0 |
| ✅ 给出正确答案，但说不出理由 | 当前深度重新问 | 维持 L | fail_count = 0 |
| ⚠️ 答案正确一半、思路对但计算错 | 肯定思路，提示计算 | 维持 L | fail_count = 0 |
| ❌ 答错（有思路但方向错）| 给提示 | 维持 L，记录 fail | fail_count += 1 |
| ❌ 连续 2 次答错 | **触发降级** | L → max(L-1, L0) | fail_count = 0 |

## 三层响应阈值

| 阈值 | 触发条件 | 响应方式 |
|:----:|----------|----------|
| **T1 轻提示** | 首次卡住（fail=1） | 缩小问题范围，不改变问题本身 |
| **T2 重提示** | 第2次卡住（fail=2，即将降级） | 换一个角度，给半成品 |
| **T3 揭示** | 降级后仍然卡住（连续4次fail） | 给完整解法，但要求孩子复述 |
