# 错题难题管理模块（子贤方法论）

> 基于子贤老师「三个什么」+ 三层诊断法 + 靶向建议

---

## ⚙️ 路径配置

错题本归档路径**不硬编码**，使用变量 `${MISTAKE_BANK_DIR}` 统一引用。

**配置规则**：

| 场景 | 路径值 | 说明 |
|:---|:---|:---|
| **默认归档目录** | `<你的工作目录>/k12-mistake-bank/` | 首次使用时询问用户指定，记住后续沿用 |
| **缓存目录** | `${MISTAKE_BANK_DIR}/.cache/` | 24h 临时缓存（家长拒绝场景） |

**AI 行为**：首次触发错题本功能时，检查路径是否存在，不存在则创建。主动询问用户指定归档目录。

## 🎯 设计目标

**两种入口**：

| 入口 | 模式 | 决策方 |
|:---:|:---:|:---:|
| 家长辅导场景 | `mode: parent-coaching` | 家长 |
| 学生自学场景 | `mode: self-study` | 学生 |

每次答对/答错/不会做之后，按当前模式主动问：

> **「这道题要加到错题本里吗？我帮你分析一下原因。」**

如果决策方说「要」或「加」，则输出**诊断卡（双格式）**：

| 格式 | 用途 |
|:---:|:---|
| 📄 Markdown | 归档存储（存入 `${MISTAKE_BANK_DIR}`） |
| 📱 HTML | 展示发送（适配手机微信，字体大、排版清晰） |

如果决策方说「不用」，走「拒绝处理流程」（见下文 4 种场景）。

## 🔍 三层诊断法（子贤方法论）

### 层次定义

| 层次 | 标签 | 颜色 | 说明 |
|:---:|:---:|:---:|:---|
| 🟡 基础技能层 | **基础技能** | 黄 `#F59E0B` | 计算错误、抄错数字、没看清条件——**会做但没做对** |
| 🟢 前置知识层 | **前置知识** | 绿 `#10B981` | 概念模糊、公式记错、知识点没掌握——**卡在「为什么」** |
| 🔴 数学思维层 | **数学思维** | 蓝 `#3B82F6` | 不会转化、不会建模、不会分类讨论——**卡在「怎么想」** |

### 前置知识层再细分

🟢 前置知识层进一步拆为 3 个子层：

| 子层 | 标签 | 说明 | 靶向建议方向 |
|:---:|:---:|:---|:---|
| 🟢-1 | **概念模糊** | 定义没理解（如：不知道「相反数」是什么） | 回教材 + 胡小群「三个什么」 |
| 🟢-2 | **公式记错/混淆** | 公式记错或用错场景（如：平方差公式 vs 完全平方公式） | 昍爸核心公式教学话术（5步法） |
| 🟢-3 | **知识点断层** | 前面该学的没学会，导致现在听不懂 | 查知识脉络，定位断层，补前置 |

## 🔄 工作流程

### 入口判断：辅导模式识别

每次辅导开始时，先识别模式（在对话中标注 `mode` 字段）：

| 模式 | 触发特征 | 决策方 | 错题本归属 |
|:---:|:---|:---:|:---|
| `mode: parent-coaching` | 家长在旁辅导，孩子答/家长转述 | 家长 | 家长决定 |
| `mode: self-study` | 学生自主学习，无家长在场 | 学生 | 学生决定 |

识别方法：
- 主动问一次：「**这次是你（家长）辅导，还是孩子自己在学？**」
- 记住答案，整个会话沿用
- 如果家长中途离开、学生接手 → 切换到 `self-study`，再次确认决策方

> **核心原则**：**谁做主，谁有决定权。** 家长辅导→家长定；学生自学→学生定。

### 拒绝处理流程

#### 4 种拒绝场景

| 场景 | 触发条件 | 决策方 | 处理策略 |
|:---:|:---|:---:|:---|
| **A. 家长拒绝** | 家长辅导后说「不用加」 | 家长 | 24h 文件缓存 + 诊断卡可外发 |
| **B. 学生拒绝** | 学生自学后说「不用加」 | 学生 | 不缓存 + 诊断卡可外发 |
| **C. 漏触发** | AI 未主动问 | 任意 | 0 打扰，不入库 |
| **D. 家长问+学生拒** | 家长帮学生问，学生不配合 | 共同 | 不替学生决定，转发诊断卡 |

---

#### 场景 A：家长拒绝（详细流程）

1. **尊重拒绝，不强推**：家长说「不用加」，不再二次推销
2. **简短询问原因**（可选）：「是因为太简单？还是觉得没时间看？」
3. **24h 文件缓存**：
   - 写入路径：`${MISTAKE_BANK_DIR}/.cache/YYYY-MM-DD-HHMM-题目关键词.md`
   - 文件头部加 `**状态：待定（家长拒绝）**` 标记
   - 24h 后自动清理（每次新增错题时扫描 .cache 目录，删除超过 24h 的文件）
4. **🟢前置知识层（断层/概念模糊）特殊处理**：
   - **强烈建议入库**（用加粗文字，不用弹窗、不用红标签）：
     > **💡 建议加入错题本**：检测到是「🟢-3 知识点断层」类问题，孩子在「为什么」这一步卡住。
     > 错题本能帮我们追踪 3 天后是否真懂。您看要不要加？
   - 仍尊重家长最终决定权
5. **生成 HTML 诊断卡**（不写入正式错题本，只生成 HTML）：
   - 路径：`${MISTAKE_BANK_DIR}/.cache/YYYY-MM-DD-HHMM-题目关键词.html`
   - 末尾标注「**未入库（家长拒绝）**」
   - 仍可发到微信，方便家长截图保存
6. **24h 内反悔**：
   - 家长说「还是加上吧」→ 把 .cache 里的 Markdown 移到正式目录 + 重命名为正式格式
   - 同步更新 INDEX.md
   - 删除 .cache 里的 HTML（重新生成标准版）

---

#### 场景 B：学生拒绝（详细流程）

1. **尊重拒绝，不强推**
2. **简短询问原因**（可选）：「觉得没必要？还是觉得太简单？」
3. **不留底**：学生拒绝 → 不写 .cache，不留任何痕迹
4. **诊断卡可外发**：
   - 仍生成 HTML（方便学生截图保存到自己的笔记）
   - 末尾标注「**未入库（学生自主决定）**」
   - 不写入 INDEX.md
5. **核心定位**：学习责任主体是学生，AI 只提供诊断服务，不强加存储

---

#### 场景 C：漏触发

如果 AI 没主动问「是否加入错题本」：
- 错题本模块不工作，零打扰
- 不入库，不记录
- 下次 AI 主动询问时再触发

---

#### 场景 D：家长问 + 学生拒

1. **不要替学生决定**：
   > 建议家长：「你可以把这个诊断卡转给他看，让他自己决定。」
2. **生成 HTML 诊断卡**（适配手机微信转发）
3. **不写入错题本**（学生未同意）
4. **学生看完后**：
   - 同意 → 正式入库
   - 拒绝 → 不入库
   - 沉默 → 7 天后自动清理（家长场景的 .cache 规则）

---

#### 拒绝处理的核心原则

| 原则 | 含义 |
|:---|:---|
| **谁做主** | 学习责任主体 = 错题本决策方 |
| **不强迫** | 拒绝就是拒绝，不二次推销 |
| **轻留底** | 家长场景：24h 文件缓存；学生场景：不留底 |
| **强建议但不入库** | 🟢断层场景必须**加粗文字**强烈建议（Q3-B） |
| **诊断卡可外发** | 拒绝后仍生成 HTML，方便转发/截图 |
| **不问就不存** | 零打扰原则 |
| **加粗不用弹窗** | 强烈建议用 **加粗文字**（Q3-B），不用 ⚠️ 弹窗、也不用红色标签 |

### Step 1：触发时机

**每次辅导结束后**（答完题、讲完解析、布置完练习），主动问：

> **「这道题要加到错题本里吗？我帮你分析一下原因。」**

触发条件：
- 孩子答错
- 孩子说「不会」
- 孩子做对了但过程磕绊
- 家长主动说「这题他老错」

**模式相关差异**：
- `mode: parent-coaching` → 问家长
- `mode: self-study` → 问学生

**触发后处理**（按拒绝处理流程）：
- 同意 → 走 Step 2-6
- 拒绝 → 走「拒绝处理流程」对应场景
  - 家长拒 → 场景 A（24h 缓存）
  - 学生拒 → 场景 B（不留底）

### Step 2：收集信息

如果家长确认加入错题本，收集：

1. **题目**（文本 / 照片描述 / 直接引用刚才讲的题）
2. **孩子的表现**（哪步卡了 / 错在哪 / 怎么错的）
3. **年级**（如果之前没说过）

### Step 3：三层诊断

按照「三层诊断法」分析：

1. **先判断主要层次**（🟡 / 🟢 / 🔴）
2. **如果是 🟢，再细分**（🟢-1 / 🟢-2 / 🟢-3）
3. **写具体诊断**（哪步卡了，为什么卡）
4. **判断次要层次**（很多时候是混合的）

### Step 4：生成靶向建议

根据主要层次，生成：

- **🟡** → 速算练习 + 检查习惯 + 家长话术
- **🟢** → 回教材章节 + 对应方法论（胡小群/昍爸）+ 家长话术
- **🔴** → 思维工具 + 引导脚本 + 延伸练习

### Step 5：双格式输出

1. **生成 Markdown 版** → 存入 `${MISTAKE_BANK_DIR}/YYYY-MM-DD-题目关键词.md`
2. **生成 HTML 版** → 同上目录，`.html` 后缀
3. **发送 HTML 到微信**（deliver_attachments）

### Step 6：跟进复习（间隔复习五步法）

> 轻量化间隔复习：三档固定周期，人工触发，AI 出题验收。

#### 6.1 入库时定复习计划

错题诊断卡生成时，**按主要层次自动决定复习周期**：

| 主要层次 | 首次复习 | 第二次 | 第三次（→ 掌握判定） |
|:---:|:---:|:---:|:---:|
| 🟡 基础技能 | **3 天后** | 7 天后 | 满足判定 → 归档 |
| 🟢 前置知识 | **5 天后** | 10 天后 | 满足判定 → 归档 |
| 🔴 数学思维 | **7 天后** | 14 天后 | 满足判定 → 归档 |

> 逻辑：基础技能主要是习惯问题，短周期刷够次数；思维层需要时间消化，拉长周期。

---

#### 6.2 到期提醒

**AI 不自动推送**（无定时器能力），由以下方式触发：

| 触发方式 | 时机 | 说明 |
|:---|:---|:---|
| **用户主动** | 任何时候 | 家长/学生说「复习一下」「检验一下」 |
| **AI 主动** | 辅导结束后 / 回答学生问题结束后 / 自学结束后 | **不在进行中打断**，只在当前事务收尾时自然提起 |

**AI 主动说的话**（轻提一句，不催、不强求）：

> **只有 1 道到期：**
> 「对了，上次那道[知识点]的题，差不多到巩固的时候了——今天顺手练一下？」

> **有多道到期：**
> 「上次我们记下了 X 道难题，好几道都到时间了，要不要趁现在一块儿过一遍？」

用户说「好」→ 进入 6.3 复习出题；说「算了」→ 跳过，不追问。

---

#### 6.3 复习出题

触发复习后，三步走：

**(a) 回顾上次诊断（口语化，不提方法名）**

> 「上次这道题，卡在公式记混了——还记得是哪个地方容易搞混吗？」

只说问题是什么、卡在哪里，**不说**「用 XX 5 步法重新记」之类的术语。

**(b) 出变式题（换场景，不是换数字）**

- 同一知识点，**换情境**：小明买苹果 → 货车运货；三角形内角和 → 四边形内角和
- 目标是孩子**不认识原题但能用同一套思路解**

**(c) 还是不会 → 先引导再降维**

| 步骤 | 做什么 |
|:---|:---|
| 先给**关键引导**（1-2 句） | 问一个切入问题，不直接给答案：「这道题你觉得第一步要算什么？」 |
| 引导后仍卡住 → **降维再出一道** | 退回更简单的情境，专练核心步骤 |

**降维逻辑**：

```
🔴 思维题卡住 → 先出一道 🟢 概念确认题（确认前置知识在不在）
🟢 概念题卡住 → 先出一道 🟡 直接套公式题（确认基础操作在不在）
```

降维后做对了 → 再回到原层次题目重试；降维后还是错 → 进入重新诊断（层次可能变了）。

---

#### 6.4 复习结果回写

| 结果 | 处理 |
|:---|:---|
| ✅ 做对，思路清晰 | 复习次数 +1，进入下一档周期 |
| ⚠️ 做对但磕绊（提示才会） | 复习次数 +1，**重置当前档周期**（再等同样时间） |
| ❌ 还是做错 | 重新诊断（层次可能变了）→ 重置，从首次复习周期重跑 |

同步更新 INDEX.md：复习次数、下次复习日期、状态。

---

#### 6.5 掌握判定 → 归档

满足以下条件 → 标记「**已掌握**」，移出活跃复习列表：

> 连续两次复习均 ✅ 做对（思路清晰，无需提示）

归档动作：
- INDEX.md 该行状态改为 `✅ 已掌握（YYYY-MM-DD）`
- Markdown 文件头部加标记 `状态: 已掌握`
- 不删除文件，留档查阅

---

#### 6.6 与 P3 机制衔接

| P3 触发时机 | 错题本做什么 |
|:---|:---|
| 每次辅导后 | 检查当日复习结果有无新失败模式，有则提炼 gotcha |
| 每月定期回顾 | 统计：新增多少 / 已掌握多少 / 哪类问题反复出现 |

## 🚨 注意事项

1. **每道题后都问**（不要等家长/学生主动说）
2. **诊断卡不带追问**（最后一条追问删掉，家长辅导建议里已有类似输出）
3. **层次标签颜色固定**（黄 🟡 / 绿 🟢 / 蓝 🔴）
4. **HTML 必须适配手机**（字体 ≥ 1em，宽度 ≤ 420px）
5. **归档路径使用 `${MISTAKE_BANK_DIR}`**（见文件头部路径配置，首次使用时会询问指定目录）
6. **24h 缓存位置**（家长拒绝场景）：`${MISTAKE_BANK_DIR}/.cache/`
7. **每次新增错题时清理 .cache**（删除超过 24h 的文件）
8. **学生自学模式必加 `mode: self-study` 标记**
9. **🟢断层场景强烈建议用加粗文字**（Q3-B 选择），不用弹窗、不用红标签
