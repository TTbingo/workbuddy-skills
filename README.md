## 这是什么？

一个 AI Agent Skill，把中国中小学数学教育领域的实战派名师——**胡小群**（原复旦附中实验班名师）、**昍爸**（中科院计算机博士）、**子贤老师**（「教练式父母训练营」积极心理学+学习策略）和**马丁·塞利格曼**（积极心理学之父，《教出乐观的孩子》《真实的幸福》）——的方法论，编码成了可执行的辅导引擎。覆盖**小学+初中+高中**全学段，支持跨阶段知识点螺旋上升衔接辅导。

它不只是"解题工具"，而是一个**教学哲学系统**：

| 不是 | 而是 |
|------|------|
| ❌ 直接给答案 | ✅ 苏格拉底式追问，引导孩子自己想出来 |
| ❌ 只讲"怎么算" | ✅ 深挖"为什么这么算"（算理 > 算法） |
| ❌ 机械刷题 | ✅ 吃透一道经典题，胜过刷 100 道 |
| ❌ 死记公式 | ✅ 从具象到抽象，理解公式的推导过程 |
| ❌ 单线结论输出 | ✅ 微信端**双发**（精简版 + 完整版 HTML），家长能直接在娃面前念 |
| ❌ 一次成型不进化 | ✅ 飞轮机制：每次真实辅导失败 → 立即追加 G 编号，Skill 越用越准 |

## 核心差异化（v2.13.0）

### 1. 教学哲学层（五条线融进行为）

- **算理优先**（胡小群）：每个概念先讲"为什么这么算"，再讲"怎么算"
- **有序枚举**（昍爸）：复杂题用递进式分类代替跳跃式猜测
- **积极心理学**（子贤老师）：情绪翻译机 + 彩虹夸夸 + 错题日复习
- **解释风格干预**（塞利格曼）：3P 诊断 + ABCDE 认知重构，避免"我永远学不会"的悲观归因
- **课标核心素养**（2022 版 + 2017 高教版）：三会用 + 十大核心素养 + 跨学段衔接

### 2. 双模式设计（自动识别身份切换）

- 🧒 **学生模式**：苏格拉底式追问，**不直接给答案**，按 T1→T2→T3 三层次递进
- 👨‍👩‍👦 **家长模式**：完整解析 + **引导脚本**（可直接照着对娃念）+ 延伸建议
- 自动识别触发词："孩子…""我家娃…"→ 家长模式；"我不会做…"→ 学生模式

### 3. 微信双发 SOP（v2.12.0+ 实战沉淀）

家长拿到辅导指南时，**微信端长消息会被截断**——本 Skill 用三件套解决：

1. 生成完整版 HTML（家长辅导指南六合一）
2. 消息 1 发**精简版**（≤1500 字，零元数据，纯内容）
3. 消息 2 用 `deliver_attachments` 单独发 HTML 附件

详见 `references/constraints-quick-ref.md` 约束 12.5-12.7。

### 4. 飞轮进化机制（v2.11.0+）

每次真实辅导失败 → 立即追加 G 编号到 `references/gotchas.md` → 升级约束 → 推 GitHub。

| 飞轮编号 | 实战场景 | 修复 |
|:----:|---------|------|
| **G25** | 双发只发"一条消息"被微信截断 | 强制分两条消息发 |
| **G26** | 说"下一条消息发"但实际没发附件 | 双发 SOP 三件套（必须 deliver_attachments）|
| **G27** | 精简版混入元数据被截断 | 精简版消息必须纯净，≤1500 字零元数据 |

### 5. 资产完整性（v2.13.0）

- **14 个 reference 资产**（方法论 4 份 / 课标 / 知识库 / gotchas / 案例 / 反例 / 4 份规范 / skill-health）

## 快速开始

### 触发方式

直接在对话中说任何与中小学数学辅导相关的内容，例如：

```
"孩子三年级，不会做鸡兔同笼，帮我讲一下"
"帮我出一道四年级的分数计算题"
"孩子计算总是出错，怎么训练？"
"这个乘法分配律为什么成立？用面积模型解释"
"中考数学压轴题，这道二次函数题怎么做？"
"高中导数的几何意义是什么？用图像解释"
"小学的正比例和高中的线性函数有什么关系？"
```

**触发词覆盖**：小学/初中/高中+数学、鸡兔同笼、奥数、算理、数感、验算、一题多解、几何辅助线、思维训练、内驱力、情绪翻译机、解释风格、ABCDE、考试挫败、数学焦虑、家长会、家校沟通、跨阶段衔接、小升初、初升高……（100+ 关键词自动识别）

**Load when 路由**（v2.11.0 升级）：
- ✅ Load when: 家长/老师辅导数学题目 或 询问辅导方法/情绪应对
- ❌ Do not load: 纯公式查询、高等数学、考研数学、奥赛冲刺、教学管理/排课/教务

### 身份识别

每次对话开始时，Skill 会先确认你是**学生本人**还是**家长/老师**，然后自动切换对应模式。

## 安全边界：这个 Skill 不会做什么

| ❌ 不会做 | 原因 |
|-----------|------|
| **直接给答案** | 核心原则：不灌答案，用启发 |
| **批评孩子** | 错了 = 发现了学习机会 |
| **超前压榨** | 四年级前不强行用抽象方式教公式 |
| **堆砌口诀** | 口诀学得越多，理解越少 |
| **写"如图"** | 没有图就绝不说"如图"，用文字引导孩子自己画 |
| **替代老师/家长** | 定位是辅导工具，不改作业、不替代教学关系 |
| **出题后直接给答案** | 默认只给思考提示，不直接提供答案 |
| **空泛安慰** | 不说"你很聪明/有天赋"，说具体行为和方法 |
| **不真发附件** | "已生成文件"≠"已发附件"，必须 deliver_attachments |

## 示例输出

### 🧒 学生模式示例

> **孩子**："鸡兔同笼，14个头，38条腿，我不会做。"

> **AI**："我们先不急着做题。你想象一下：如果笼子里全是鸡，那14只鸡有多少条腿？"

> **孩子**："28条。"

> **AI**："对！但题目说有38条腿，差了多少条？"

> **孩子**："差了10条。"

> **AI**："很好。那每把一只鸡换成一只兔子，腿会多几条？"

> **孩子**："……多2条！"

> **AI**："所以要补上10条腿，需要换几只？"

> **孩子**："5只！所以兔子5只，鸡9只！"

### 👨‍👩‍👦 家长模式示例（含微信双发）

**消息 1（精简版 ≤ 1500 字）**：

> 📋 **家长辅导指南 · 三步归一 · 惠民购物（432 瓶）**
>
> **题目**：便利店进货 24 瓶/箱 × 18 箱，促销每 6 瓶一组，1 组售价 72 元，**共可卖多少元？**
>
> **答案速查**：864 元
>
> **算理解析**：分三步——先算总瓶数（24×18=432）→ 再算组数（432÷6=72）→ 再算总价（72×72=864）
>
> **常见错误**：
> - ❌ 第一步用错运算（用"24×18"做减法）
> - ❌ 第二步忘记单位换算（瓶→组）
> - ❌ 第三步把"72 组"和"72 元/组"混淆
>
> **引导脚本**（您对娃念）：
> 1. "一箱有 24 瓶，18 箱呢？先算第一小题。"
> 2. "6 瓶是一组，432 瓶能拼成多少组？"
> 3. "一组卖 72 元，72 组呢？"
>
> **知识脉络**：整百整十数乘除 → 乘除混合 → 归一问题
>
> **延伸练习**：把 24 瓶/箱改成 12 瓶/箱，看解题步骤怎么变？
>
> 完整版（含图示+变式训练+情绪预判）见附件 ↓

**消息 2（完整版 HTML 附件）**：📎 家长辅导指南_三步归一_惠民购物_v2.11.0.html
